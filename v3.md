# TECH SPEC V3: Dynamic Backfilling & Strategic Pullback

## 1. STATE MANAGEMENT & SLOTS (`riskManager.ts`)

**Constraints:**

- `MAX_GLOBAL_POSITIONS` = 5
- `TOTAL_CAPITAL` = Baseline (e.g., $100,000)
- `SATELLITE_ENVELOPE` = 20% of `TOTAL_CAPITAL`
- `CORE_ENVELOPE` = 80% of `TOTAL_CAPITAL`

**Time-Decay Slot Logic:**

- @ 09:30 EST -> maxCore: 4, maxSatellite: 1
- @ 10:15 EST (+45m) -> IF activeCore < 4 THEN maxCore: 3, maxSatellite: 2
- @ 11:00 EST (+90m) -> IF activeCore < 3 THEN maxCore: 2, maxSatellite: 3
- @ 11:45 EST (+135m) -> IF activeCore < 2 THEN maxCore: 1, maxSatellite: 4

**Watchlist Persistence (`index.ts` / `premarket_screener.ts`) :**

- STRICT RULE: Do not purge or overwrite the `V2_PLAYMAKER` targets from `watchlist.json` or memory after the morning scan.
- The 10 Satellite candidates MUST remain active in the monitoring loop for the entire trading session to catch late pullbacks.

**Risk Bleed Prevention:**

- IF a `V2_PLAYMAKER` (Satellite) signal uses a backfilled Core slot, its `qty` calculation MUST strictly use `SATELLITE_ENVELOPE / activeSatelliteSlots` and 1% risk of that sub-envelope. It never inherits Core capital.

## 2. PULLBACK ENTRY MICRO-STRUCTURE (`trader.ts`)

**State Machine for Signals:**
`PENDING` -> `TRACKING_PULLBACK` -> `TRIGGERED` -> `EXECUTED`

**Transition Rules:**

1. `PENDING` to `TRACKING_PULLBACK`:
   - Condition: `currentPrice > VWAP` AND `RVOL >= 1.2`
   - Action: Log local high, wait.
2. `TRACKING_PULLBACK` to `TRIGGERED`:
   - Condition: Price retraces to dynamic support.
   - Formula: `Math.min(abs(price - VWAP), abs(price - EMA9)) <= (price * 0.002)`
   - Action: Wait for first Tick UP (price increases compared to previous tick).
3. `TRIGGERED` to `EXECUTED`:
   - Action: Send Alpaca limit order.

## 3. ORDER EXECUTION & RACE CONDITIONS

**Alpaca 422 Error Prevention:**

- Always use `Marketable Limit Order` for entries.
- `limitPrice = parseFloat((currentAsk * 1.001).toFixed(2))`
- Send as `order_class: 'oto'` with strictly formatted `stop_loss`.

**Concurrency:**

- Lock the `flushPendingSignals` loop.
- Priority: If `V1_CORE` and `V2_PLAYMAKER` trigger simultaneously, `V1_CORE` consumes the slot first.
