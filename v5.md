# TECH SPEC V5: Trading Journal & Continuous Feedback Loop

## 1. Objectif Architectural

Créer un module d'apprentissage (`journalManager.ts`) pour tracker, analyser et archiver le cycle de vie complet de chaque transaction. Le but est de déterminer pourquoi un trade a réussi ou échoué en croisant les données du screener avec l'exécution en temps réel.

## 2. Structure de Données (Le Journal)

Créer une interface `TradeRecord` qui stockera l'historique dans un fichier local `data/journal.json` (ou une base de données type SQLite/Prisma si disponible).
Chaque enregistrement doit contenir :

**A. Contexte Pre-Trade (Screener Data) :**

- `symbol` (ex: QBTS)
- `origin` (V1_CORE ou V2_PLAYMAKER)
- `alpha_vs_spy` (Performances relatives)
- `gap_percentage`
- `relative_volume` (RVOL)

**B. Contexte d'Entrée & Indicateurs Techniques (Execution Data) :**

- `entry_time` et `entry_price`
- `vwap_at_entry` (Prix par rapport au VWAP : au-dessus/en-dessous)
- `ema9_at_entry` (Moyenne Mobile Exponentielle 9 - Court terme)
- `sma20_at_entry` (Moyenne Mobile Simple 20 - Support dynamique)
- `distance_to_sma20_percent` (L'actif est-il trop étiré par rapport à sa MM20 ?)
- `spy_trend_5m` (Le marché global SPY était-il haussier ou baissier au moment de l'achat ?)

**C. Métriques Quantitatives de Sortie (Resolution Data) :**

- `exit_time` et `exit_price(s)`
- `exit_reason` (Target 5%, Target 7%, Stop Loss initial, Trailing Stop, EOD Liquidation)
- `net_pnl_dollars` et `net_pnl_percentage`
- `MFE_percent` (Maximum Favorable Excursion : le gain maximum latent atteint pendant le trade)
- `MAE_percent` (Maximum Adverse Excursion : la perte maximum latente subie pendant le trade)

## 3. Logique de Capture

- **À l'achat :** `trader.ts` initialise un `TradeRecord` en mémoire avec les données A et B.
- **À la vente :** Le système complète les données C et pousse l'objet complet dans le fichier `journal.json`.

## 4. Module d'Analyse (Post-Mortem EOD)

Créer un script `analyzer.ts` qui s'exécute automatiquement après le "Hard Close" (15h58 EST).
Ce script doit calculer les KPI quotidiens suivants à partir du journal :

- **Win Rate :** % de trades avec PnL positif.
- **Average Win vs Average Loss :** Le ratio de profitabilité.
- **Best Setup :** Identifier quel critère (ex: RVOL > 2, Origin = V2) a généré le meilleur PnL aujourd'hui.

## 5. Notification V5

Intégrer le module V4 (`notificationManager.ts`) pour envoyer ce rapport EOD directement sur Telegram sous forme de résumé analytique :
`📊 <b>[ANALYSE V5]</b> Win Rate: 65% | Meilleur setup: V2 avec RVOL > 3x (QBTS)`
